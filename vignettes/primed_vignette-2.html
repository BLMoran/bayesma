Introduction

Meta-analyses of dependent effect sizes routinely involve complex hierarchical data — multiple effect sizes nested within samples, samples nested within studies, moderator variables at several levels, and auxiliary statistics like standard errors that behave differently across effect size metrics. Before fitting any formal model, a careful analyst needs to understand what’s actually in the database.

The PRIMED workflow (PReliminary Investigation of MEta-analytic Databases) proposed by Pustejovsky, Zhang, and Tipton (2026) organises this preliminary work into four sequential steps:

  1. Describe the amount of data and dependence structure
  2. Explore study characteristics and potential moderators
  3. Inspect standard errors and other auxiliary data
  4. Visualise the distribution of effect size estimates

Crucially, the effect size distribution is examined last, which preserves some room to refine modelling decisions without being influenced by the results themselves.

This vignette demonstrates the primed() function, which automates the workflow — producing the summary tables and visualisations for each step in a single call. We illustrate using a synthetic meta-analytic dataset that mimics the structure of a psychosocial intervention review.

A synthetic meta-analytic database

We simulate a dataset resembling a meta-analysis of psychosocial interventions with 30 studies and around 50 samples contributing roughly 200 dependent effect sizes (SMDs). The dataset includes features typical of social-science meta-analyses: some studies contribute multiple participant samples, most samples contribute multiple effects (measured at different follow-up times or using different outcome instruments), and several moderators have missing values.

  mean_age = round(rnorm(n_studies, mean = 52, sd = 12), 1),
  pct_female = pmin(100, pmax(0, round(rnorm(n_studies, 65, 22)))),
  intervention = sample(
    c("CBT", "Mindfulness", "Education", "Creative Arts", "Multimodal"),
    n_studies, replace = TRUE,
    prob = c(0.30, 0.25, 0.20, 0.15, 0.10)
  ),
  control_type = sample(
    c("Standard Care", "Attention Control"),
    n_studies, replace = TRUE,

  # Introduce missingness in mean_age (~10%) and pct_female (~7%)
  dplyr::mutate(
    mean_age = dplyr::if_else(runif(n_studies) < 0.10, NA_real_, mean_age),
    pct_female = dplyr::if_else(runif(n_studies) < 0.07, NA_integer_, pct_female)
  )

samples <- studies |>
  tidyr::uncount(n_samples) |>
  dplyr::group_by(study) |>
  dplyr::mutate(
    sample_id = paste0(study, "-S", dplyr::row_number()),
    n_total = sample(30:250, dplyr::n(), replace = TRUE)
  ) |>
  dplyr::ungroup()

n_total_samples <- nrow(samples)

samples <- samples |>
  dplyr::mutate(
    n_effects = sample(
      1:8, n_total_samples, replace = TRUE,
      prob = c(0.15, 0.25, 0.20, 0.15, 0.10, 0.07, 0.05, 0.03)
    )
  )

meta_df <- samples |>
  tidyr::uncount(n_effects) |>
  dplyr::group_by(sample_id) |>
  dplyr::mutate(
    es_id = dplyr::row_number(),
    outcome_measure = sample(
      c("FACIT-Sp", "FACIT-Sp-Ex", "WHOQOL", "SF-36", "Custom"),
      dplyr::n(), replace = TRUE,
      prob = c(0.40, 0.20, 0.15, 0.15, 0.10)
    ),
    weeks_follow_up = sample(
      c(4, 8, 12, 16, 24, 36, 52),
      dplyr::n(), replace = TRUE
    )
  ) |>
  dplyr::ungroup()

study_effects <- studies |>
  dplyr::select(study) |>
  dplyr::mutate(theta_study = rnorm(dplyr::n(), mean = 0.25, sd = 0.30))

meta_df <- meta_df |>
  dplyr::left_join(study_effects, by = "study") |>
  dplyr::group_by(sample_id) |>
  dplyr::mutate(
    theta_sample = theta_study + rnorm(1, 0, 0.10),
    se = sqrt(4 / n_total + 0.02),
    df = n_total - 2,
    yi = theta_sample + rnorm(dplyr::n(), 0, 0.15) + rnorm(dplyr::n(), 0, se)
  ) |>
  dplyr::ungroup() |>
  dplyr::select(
    study, sample_id, es_id, yi, se, df, n_total,
    intervention, control_type, outcome_measure,
    mean_age, pct_female, weeks_follow_up
  )

# Plant one obvious outlier for diagnostic interest
meta_df <- meta_df |>
  dplyr::mutate(
    yi = dplyr::if_else(study == dplyr::first(study) & es_id == 1, 2.1, yi)
  )

Running the workflow

A single call to primed() runs all four steps:

Code 
results <- primed(
  data        = meta_df,
  es_col      = "yi",
  se_col      = "se",
  study_col   = "study",
  sample_col  = "sample_id",
  n_col       = "n_total",
  moderators  = c("intervention", "control_type", "outcome_measure",
                   "mean_age", "pct_female", "weeks_follow_up"),
  es_type     = "SMD",
  df_col      = "df",
  rho_values  = c(0.3, 0.5, 0.7)
)

print(results)

Diagonal cells report the total number of studies (and effects) for each level. Off-diagonal cells with non-empty values represent studies that include both levels — i.e., within-study variation. Completely empty off-diagonals mean the moderator has no within-study variation, so its explanatory power comes entirely from between-study differences (which are confounded with other study characteristics).

Step 3: Standard errors and weights

Step 3 focuses on the auxiliary statistics that drive modelling but aren’t the primary quantity of interest.

Standard errors by sample

Code 
results$plots$step3_se_by_sample

What to look for: rows with wide scatter warrant investigation. Common causes include multi-arm trials with unequal \(n\) per arm, differential attrition across follow-ups, or varying missing data across outcome measures.

Scaled standard errors

The SE of a standardised mean difference depends on the SMD itself, inducing artefactual variation. The scaled SE removes this:

\[ SE_{\text{scaled}} = \sqrt{SE^2 - \frac{d^2}{2\nu}} \]

Code 
results$plots$step3_scaled_se

ISC weights

The inverse sampling covariance (ISC) weights show how much each sample contributes to the overall average under a simplified model (no heterogeneity beyond sampling error), as a function of the assumed within-sample correlation \(\rho\):

\[ w_j = \frac{k_j}{\sigma_j^2 \left[(k_j - 1)\rho + 1\right]} \]

Code 
results$plots$step3_isc_weights

What to look for:

  • Samples soaking up >15–20% of total weight — plan sensitivity analyses that drop them
  • Weights that shift substantially with \(\rho\) — results may be sensitive to the assumed correlation structure

Outliers are flags for investigation, not automatic removal. Check the underlying data: is the value a coding error (wrong sign, decimal misplacement)? A metric mismatch (correlation reported where SMD expected)? Or a genuinely unusual study that is important to report on?

Hierarchical forest plot

Code 
results$plots$step4_forest

Unlike a conventional forest plot, effects are grouped on a single row per sample. This lets you see:

  • Marginal outliers — points far from the overall distribution (look for the planted outlier at \(g \approx 2.1\))
  • Within-sample outliers — points discrepant from their own sample’s other estimates
  • Heterogeneity sources — heavy between-sample spread + tight within-sample clusters suggests a two-level model suffices; spread at both levels calls for a three-level model or robust variance estimation

We examined the distribution of effect sizes across studies and samples (Step 1), the marginal and hierarchical structure of candidate moderators (Step 2), the distribution of standard errors and inverse sampling covariance weights under assumed within-sample correlations of \(\rho = 0.3, 0.5, 0.7\) (Step 3), and the distribution of effect size estimates at both the effect and sample levels (Step 4). Summary

The primed() function bundles a standard set of preliminary analyses for meta-analyses of dependent effect sizes into a single call, producing the tables and plots recommended in Pustejovsky, Zhang, and Tipton (2026). The intended workflow is:

  1. Run primed() on your cleaned database
  2. Work through steps 1–3, making and documenting any coding/moderator decisions
  3. Only then examine step 4 output
  4. Fit your formal meta-analytic model
  5. Archive the PRIMED output alongside your analysis code

Sharing these outputs makes the pre-modelling phase of meta-analysis visible and reproducible — something the field has historically under-documented.
